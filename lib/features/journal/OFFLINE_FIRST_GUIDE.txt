// Configuration guide for Journal Offline-First optimization

// ═══════════════════════════════════════════════════════════════════
// 1. OPTIMISTIC LOADING — JournalNotifier
// ═══════════════════════════════════════════════════════════════════

// Location: lib/features/journal/presentation/providers/journal_provider.dart
//
// Changes:
// • upsert() updates UI state immediately before Firestore sync
// • delete() removes from UI immediately, syncs async in background
// • .catchError() silently logs failures (TODO: add toast notifications)
//
// Effect: No loading spinner, user sees instant feedback

// ═══════════════════════════════════════════════════════════════════
// 2. HIVE CACHING — JournalFirestoreRepository
// ═══════════════════════════════════════════════════════════════════

// Location: lib/features/journal/data/repositories/journal_firestore_repository.dart
//
// Key methods:
// • initHive() — Opens or retrieves Hive box 'journal_entries'
// • _loadFromHive() — Reads all entries from cache (synchronous)
// • _saveToHive() — Persists entries to Hive
//
// Flow:
//   1. load() → read Hive cache instantly
//   2. Firestore fetch in background
//   3. On success → sync Hive + update state
//   4. On error → return cache
//
// Result: App never shows blank screen, always has cached data

// ═══════════════════════════════════════════════════════════════════
// 3. SKELETON LOADING — JournalSkeleton Widget
// ═══════════════════════════════════════════════════════════════════

// Location: lib/features/journal/presentation/widgets/journal_skeleton.dart
//
// Features:
// • Shimmer animation (requires: flutter pub add shimmer)
// • 3-4 fake cards with correct layout
// • 12px border radius matching design
// • Shows while initial load from Hive happens
//
// Usage in JournalScreen:
//   loading: () => JournalSkeleton(count: 4),

// ═══════════════════════════════════════════════════════════════════
// 4. SKIPLOADINGONRELOAD — UI Non-Blocking Refresh
// ═══════════════════════════════════════════════════════════════════

// Location: lib/features/journal/presentation/screens/journal_screens.dart
//
// Code:
//   final state = ref.watch(journalProvider.select(
//     (s) => s.skipLoadingOnReload,
//   ));
//
// Effect:
// • Refresh from Firestore doesn't trigger loading state
// • UI stays responsive with cached data
// • New data appears when ready

// ═══════════════════════════════════════════════════════════════════
// 5. UX OPTIMIZATIONS
// ═══════════════════════════════════════════════════════════════════

// A. Save Flow (JournalEditorScreen._save):
//    1. User taps "Save"
//    2. Entry added to UI immediately via upsert()
//    3. Screen navigates back to list
//    4. User sees their new entry instantly
//    5. Hive persists in background
//    6. Firestore syncs when ready
//
// B. Delete Flow:
//    1. User taps delete
//    2. Entry removed from UI list immediately
//    3. Hive & Firestore cleanup async
//
// C. Refresh Flow:
//    1. Pull-to-refresh calls load()
//    2. State shows cached data + shimmer for new items
//    3. Firestore updates appear seamlessly
